# Scratch Canvas — Idea One-Pager

## HMW

사용자가 결과를 예측할 수 없는 상태로 시작하되,
긁어낼수록 자신만의 작품이 드러나는 발견의 쾌감을 디지털로 재현할 수 있을까?

## 문제

물리 스크래치 아트의 핵심 쾌감(긁어서 발견)은 키트가 있어야 하고, 한 번 쓰면 끝이다.
디지털 도구는 스크래치 특화가 없다.

## 솔루션

브라우저에서 즉시 사용, 색상 레이어 위에 검은 오버레이를 덮고
마우스/터치로 긁어내는 힐링 경험.

색상 레이어는 두 가지 방법으로 준비한다:

1. **프리셋 선택**: 미리 만들어진 6종 색상 레이어 (그라데이션, 무지개, 야경 등)
2. **직접 칠하기**: 흰 바탕에 크레파스 16색으로 직접 색을 칠해 나만의 레이어 제작

## 사용자

힐링/취미를 원하는 성인. 혼자 즐기는 개인 경험.

## MVP Scope

- [ ] 색상 레이어 준비 단계
  - 프리셋 6종 선택
  - 흰 캔버스에 크레파스 16색으로 직접 칠하기
- [ ] 스크래치 단계: 검은 오버레이 + 브러시로 긁기 (사이즈 조절)
- [ ] 완성 후 PNG 저장

## Not Doing (MVP)

- 로그인/계정
- 공유/SNS 기능
- AI 이미지 생성
- 실시간 협업
- 커스텀 이미지 업로드로 색상 레이어 만들기

## 가장 위험한 가정

**Must Be True**: Canvas API로 자연스러운 긁기 느낌 구현 가능 (끊김 없는 브러시)

**Should Be True**: 직접 색을 칠한 레이어를 긁을 때 프리셋보다 더 큰 성취감을 준다

**Might Be True**: 크레파스 16색이 충분하다 (더 많은 색을 원하지 않는다)

## 기술 메모

- 핵심: `Canvas API` + `globalCompositeOperation: 'destination-out'`
- 색상 레이어 칠하기: 동일한 Canvas에 브러시 도구
- 크레파스 질감: 브러시 불투명도/노이즈로 표현 가능
- 플랫폼: 웹 (브라우저), 마우스 + 터치 지원
